Git 命令

git 使 .gitignore生效

git rm -r --cached .
git add .
git commit -m 'update .gitignore'

命令梳理

http://gogojimmy.net/2012/01/17/how-to-use-git-1-git-basic/

git 基本功

status

git status

查看git当前的状态： 所在分支，有没有需要add的，需不需要push到远程仓库
On branch master
Your branch is up-to-date with ‘origin/master’.
nothing to commit, working directory clean

add

git add .

添加全部未添加到本地仓库的文件。后面跟文件名表示添加某个文件。

commit

git commit

弹出编辑器，填写提交信息，按ZZ保存退出，按ZQ不保存退出

快速提交：

git commit -m '提交的信息'

在commit前需要add修改的文件，使用-am可以简化add，但是不推荐

git commit -am "Add test.rb to test git function"

git branch

每次开发一个新功能或者调整一个bug的时候要先从主干或者开发分支中拉出一条分支，代码编程完成后合并；

图形界面查看分支情况

gitk --all

查看当前分支

同时也会列出当前项目中所有的分支

git branch

创建一个分支

git branch [name]

切换分支

git checkout [branch name]

整理branch – rebase

git rebase master

会把master单前的状态从新加载到当前分支中；

合并branch

git merge [name]

合并到分支中

##　远程仓库

git remote add [shortname] [url]

查看当前仓库：

$ git remote
origin
$ git remote -v
origin  git://github.com/schacon/ticgit.git
pb  git://github.com/paulboone/ticgit.git

添加远程仓库：

$ git remote add pb git://github.com/paulboone/ticgit.git

push

强行push：pd为远程仓库的短命名，一般为origin

git push pd master

重命名 git 远程仓库名

git remote rename [oldname] [newname]

重命名 git 本地分支

git branch -m [newname]
