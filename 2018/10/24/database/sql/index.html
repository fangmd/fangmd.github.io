多表之间的关系如何维护：

  1. 外键约束
    alter table product add foreign key(cno) references category(cid);
  2. 多表之间的建表原则
    一对多：多的一边加外键
    多对多：通过中间表维护关系，中间表至少有两个外键分别指向两个表。
    一对一：一张表维护；两张表使用同一个 id 作为主键；使用外键
    拆表操作：将常用信息和不常用信息拆开，减少臃肿
